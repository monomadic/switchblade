# Selection, randomizers & file actions

These notes are the durable record of how the subsystem works **and of the bugs that shaped it** — the war stories are load-bearing, not history. Read the relevant section before changing this area.

## Random / shuffle / auto-skip (`x` / `s` / `t`)

Both randomizers are **cached-only** — `Clip.cached` flips true when the clip's gen-sweep job reports done or any artifact delivers (GenDone carries the path), so landing on a clip never queues on-demand ffmpeg work; the pool widens as the sweep runs. `jump_random` picks a uniformly random other *cached* clip (no-op while none are); `shuffle_library` Fisher–Yates the **cached clips to the front** of the library (unswept clips keep their order after them) and **remaps every index-keyed reference through the permutation** (path→index map, atlas slot owners, live/warm/hover lane `clip` fields, the selection — the selected clip keeps playing and lands somewhere new; hover clears; reflow crossfade covers the rearrange; mid-ingest arrivals append after the shuffled block); **auto-skip** (`toggle_auto_skip`, `auto_skip_s` tuning; `toggle_skip_timer`/`skip_timer_s` stay as legacy config aliases) auto-advances selection once the clip has *played* `auto_skip_s` — but ONLY while quickview or fullview is up: back in the grid the countdown suspends, and any suspension (grid, paused, parked, scrubbing, mid-swap, chapter bar) **re-anchors** `auto_skip_since` every tick so re-entering a modal always starts a fresh countdown, never fires instantly on stale watch time. Countdown starts at first live frame (falls back to selection age + a 3s stall grace when a stream never delivers, plain selection age at non-live animation levels) and resets whenever the selection changes; the advance is the same linear "next" the `next` command performs (wrapping at the end). While the countdown exists, a **timer ring** — a stroked white arc over a soft dark scrim, anchored to the VIDEO's top-right corner (`active_video_rect`; window corner while nothing's decoded): by default it FILLS clockwise from 12 toward the next clip (count-up; `auto_skip_countdown = true` flips to a draining countdown), sized by `auto_skip_ring_radius` tuning (`push_countdown_ring`, gated on `auto_skip_progress()`) — shows it; it vanishes with auto-skip off or the modal closed. The arcs are `Tile.pie` (0 = normal tile; else the shader clips fill AND border — so a border-only circle becomes an arc — to a wedge of |pie|: positive ends at 12 and drains clockwise, negative starts at 12 and grows) — the one non-rect thing tiles.wgsl draws, added for this and reusable for any radial meter. Randomness is std-hasher-seeded xorshift — no rand dependency. Tests: `shuffle_remaps_every_index_keyed_reference`, `auto_skip_advances_and_wraps`, `jump_random_lands_only_on_cached_clips`.

## Drag-out (grid tiles + filmstrip chips)

Pressing a tile/chip and pulling past `drag_threshold` (tuning, logical px) turns the press into a native macOS drag — `WindowCommand::BeginDrag {path, image}` crosses the App boundary and sb-window's `drag.rs` starts an `NSDraggingSession` whose pasteboard carries the file URL, so dropping into Finder/other apps behaves exactly like dragging out of Finder. The ghost image is the clip's cached thumb jpeg (`MediaService::cached_thumb_path`), falling back to the OS file icon. Two load-bearing rules: the command must be drained **inside the CursorMoved callback** (sb-window calls `apply_commands` there) because the session is seeded from `NSApp.currentEvent`, which must still be the live mouse-drag; and **quickview-open moved from mouse-down to mouse-up** (`Press.open_quickview`) so dragging the selected tile can't open the modal — a matured drag clears the press (the OS owns the gesture; no MouseUp may arrive). Chips select on press as before, then drag the selected clip. Test: `tile_drag_out_matures_and_click_still_opens_quickview`.

## Siblings swap (`D`, `open_parent`)

Rebuilds the library from the selected clip's parent dir (non-recursive `spawn_dir_reader`), streamed like any ingest. `pending_reselect` holds the clip's path across the swap: update_live neither reaps nor spawns by index while it's set (indices are churning; the shielded stream keeps playing — display gates match by **path**, not index), and when the path streams back in it becomes the selection again (cleared on ingest close if the file vanished). Old jobs keep counting; thumbs re-serve from the disk cache. Test: `open_parent_swaps_to_siblings_and_reselects`.
