# Media pipeline, ingest & GPU residency

These notes are the durable record of how the subsystem works **and of the bugs that shaped it** тАФ the war stories are load-bearing, not history. Read the relevant section before changing this area.

## Media pipeline

ffmpeg/ffprobe workers behind a **strict four-tier queue** тАФ visible thumbs тЖТ meta reprobes (one cheap ffprobe rewrites a pre-`pix_fmt` meta.json so that clip's next live spawn can go hardware тАФ the cache never expires, so old entries heal lazily; never probe on the render thread) тЖТ the on-demand storyboard sheet (`request_anim_now`, one clip тАФ the user is in quickview/fullview NOW; it originally queued in a bulk-anim tier below gen, and a thumb-recipe change backed gen up for DAYS, silently starving the storyboard тАФ user-attention work never waits behind a sweep) тЖТ library-wide background thumb gen (every ingested file queues a disk-cache-only gen job; the progress bar tracks it). **There is no bulk sheet tier anymore** (removed 2026-07-20, DESIGN.md Level 2 note): sheets generate ONLY on demand when quickview/fullview opens a clip тАФ clips never opened never pay for one, and the freed workers finish the thumb sweep (and widen the shuffle/random pool) sooner. (Workers use `-hwaccel videotoolbox` for h264/hevc/prores; anim sheets are g┬▓ *keyframe-grabbed single-frame extracts* (`-noaccurate_seek` тАФ the nearest keyframe to each k/n timestamp, no decode-forward, one process, sw decode) + one tile pass тАФ NEVER a full-clip `fps=` decode, which cost minutes of multi-core churn per long 4K source and survived app quit) тЖТ sidecar cache under `~/Library/Caches/switchblade.noindex/v1/objects/<fp>/` (`.noindex` keeps Spotlight's mdworker off the thousands of jpegs; an old un-suffixed cache is renamed across on startup) (artifacts named by recipe with the *ffmpeg* -q:v value; config `thumb_quality` is 1..10, mapped `-q:v = 12 - q` тАФ e.g. `thumb_fit_640x360_q5_kf.jpg`, `anim_3x3_640x360_q5_fit_kf.jpg` + `meta.json`; anim cells preserve the source's TRUE aspect тАФ fit into the thumb/grid cell box, NOT crop-filled to 16:9 тАФ the `_fit` suffix regenerates the old cropped sheets, and `_kf` regenerates the exact-frame thumbs/sheets from before the `-copyts` fix below) тЖТ decoded to RGBA **via ffmpeg** (not the image crate тАФ keeps thumb and live-video color math identical; the image crate is header-dims only) тЖТ runtime-sized GPU atlas (`AtlasCfg`, from `thumb_width/height` + `atlas_width/height` tuning, startup-only; тЙд16384/side тАФ device limits are requested to match, don't assume wgpu's 8192 default).

## Ingest gatekeeper (validation + optional ordering, 2026-07-23)

Two stages, neither on the render thread. Stage one (ingest thread, `gatekeeper` config, default on): zero-byte files and files whose first 16 bytes don't match their extension's container family are rejected before they claim a chip (`plausible_video` тАФ one tiny read, never a probe; container families without a confident signature fail open; cloud placeholders are never read). Stage two: when ffmpeg itself fails to pull a frame from the source (corrupt/truncated body behind a valid header, or the file vanished), the job emits `ThumbResult::Unplayable` тАФ in place of its final Failed/GenDone so the jobs ledger still balances; missing-ffmpeg and cache-artifact decode failures stay plain `Failed` (no verdict, clip stays) тАФ and the app drops the clip from the grid (`remove_clip` тЖТ `remap_refs`, the remap factored out of shuffle: pathтЖТindex map, atlas slot owners, live/warm/hover lanes, selection-to-nearest-survivor, marks, and the reflow table PERMUTED so neighbors glide over the hole instead of snapping). **Sorted ingest** (`--sort newest|oldest` CLI / `sort` config, startup-only): arrivals merge into a creation-date-sorted grid AS THEY STREAM тАФ the ingest thread's existing stat carries `created` (birthtime, mtime fallback) on `Ingested`, and `install_arrivals` does ONE stable merge + ONE remap per drained frame, never per item (a per-item shift is quadratic over a big stream); surviving tiles keep their reflow entries, so a late-arriving out-of-order file pushes the grid aside with a glide, and the selection follows its clip through every remap. A shuffle disarms sorted merging for the session (the arrangement is the shuffle's; arrivals append after it, as ever); a D-swap/`open_library` rebuild re-arms it. Deliberately distinct from M9's planned runtime sorts (those are a VIEW over the sacred ingest vector; this is at-open ordering тАФ under `--sort` the original stdin order is not preserved). Tests: `garbage_behind_a_video_extension_never_reaches_the_grid` (ingest), `sorted_ingest_places_arrivals_by_created_date`, `unplayable_clip_is_removed_from_the_grid`.

## GPU-residency drops are retryable

When the atlas is momentarily full, arriving thumbs/sheets are dropped back to `None` (never latched `Failed`) so they reload when visible тАФ the disk cache (unlimited, nothing expires) makes retries cheap.

## Slot budget & eviction classes

Statics claim budget in request-priority order тАФ the selected tile first, then on-screen rows top to bottom, then the prefetch margins (below, then above); anim (storyboard) slots only ever appear from on-demand quickview/fullview installs, a handful at most. (This replaced a center-out walk from the viewport's middle, 2026-07-23: meant as "load where the eye rests", it read as broken on open тАФ the grid filled from mid-screen while the first rows and the selected tile sat empty.) Eviction order = out-of-zone anims тЖТ out-of-zone statics тЖТ in-zone anims; never in-zone statics or live slots.
