# Switchblade — agent notes

GPU-rendered video clip picker: "fzf for videos". Pipe paths in on stdin, fly a grid of clips, act on the selected one.

**[PLAN.md](PLAN.md) is the source of truth** for scope, milestones, and every design decision. Read the relevant section before structural changes; don't re-litigate settled decisions (torus deferral, filesystem cache over SQLite, animated thumbs as MVP v2, etc.) without the user.

## Status
- **Done:** M0–M6 (see PLAN.md §14) plus the internal quickview (M5's "later" form): `Space` or clicking the selection opens a modal playing the clip **at natural resolution** (the same hires stream the selected tile plays — see below), with a **filmstrip** of neighbors along the bottom (selected centered at `strip_selection_scale`, hovered chips scale to `strip_hover_scale` and play their own tile-size lane; both draw above neighbors like the grid; slides on `strip_snap_strength`, chips clickable; `strip_height`/`strip_gap`). In quickview, actions target the foreground: `h`/`l` advance clips, clicking a chip selects it, any other click / Esc / Space closes. The grid is backdrop: dimmed (`quickview_dim`) and **frosted** — the grid layer renders offscreen, walks a mip chain, and draws back blurred (`quickview_blur` = downsample levels, 0 disables; `Frame.blur` carries the split across the App boundary). **Neighbor decoders pre-warm** for the four movement destinations (right, right+1, down, left) in quickview AND in the grid (when live video is on): `warm` holds parked-by-backpressure players, the outgoing stream demotes to warm instead of dying, promotion serves a queued frame the same tick — so single-step selection moves show video instantly. Two ordering rules, both load-bearing: promotion MUST run before the pool is pruned to the new destinations (the selection is never its own neighbor; pruning first killed the warmed decoder and made every advance pay full cold-spawn latency, ~1s on 4K — a cold spawn's floor is probe+VideoToolbox init+first GOP, benchmarked unfixable via flags); and fresh warm spawns wait for settle + the selected stream's first frame, then run one at a time (user attention owns the CPU).
- **GPU-residency drops are retryable:** when the atlas is momentarily full, arriving thumbs/sheets are dropped back to `None` (never latched `Failed`) so they reload when visible — the disk cache (unlimited, nothing expires) makes retries cheap.
- **Media pipeline:** ffmpeg/ffprobe workers behind a **strict four-tier queue** — visible thumbs → meta reprobes (one cheap ffprobe rewrites a pre-`pix_fmt` meta.json so that clip's next live spawn can go hardware — the cache never expires, so old entries heal lazily; never probe on the render thread) → library-wide background thumb gen (every ingested file queues a disk-cache-only gen job; the progress bar tracks it) → anim sheets, so every file has a thumbnail before ANY sheet work starts and newly discovered files always jump ahead of sheets (workers; workers use `-hwaccel videotoolbox` for h264/hevc/prores; anim sheets are g² *seeked single-frame extracts* + one tile pass — NEVER a full-clip `fps=` decode, which cost minutes of multi-core churn per long 4K source and survived app quit) → sidecar cache under `~/Library/Caches/switchblade.noindex/v1/objects/<fp>/` (`.noindex` keeps Spotlight's mdworker off the thousands of jpegs; an old un-suffixed cache is renamed across on startup) (artifacts named by recipe with the *ffmpeg* -q:v value; config `thumb_quality` is 1..10, mapped `-q:v = 12 - q` — e.g. `thumb_fit_640x360_q5.jpg`, `anim_3x3_640x360_q5.jpg` + `meta.json`) → decoded to RGBA **via ffmpeg** (not the image crate — keeps thumb and live-video color math identical; the image crate is header-dims only) → runtime-sized GPU atlas (`AtlasCfg`, from `thumb_width/height` + `atlas_width/height` tuning, startup-only; ≤16384/side — device limits are requested to match, don't assume wgpu's 8192 default).
- **Live playback:** the selected clip decodes **once** at quickview resolution (`quickview_max_*`, capped at source res) into the mipmapped hires texture — the tile samples it downscaled, quickview samples it big; one decoder, one timeline, so quickview opens instantly with no handoff/skip. Runs at animation level `normal`+ (quickview skips the settle delay). **Animation levels** (`animation` config / `--animation` CLI, default `normal`): `none` = snap-everything (every spring/fade completes in one frame), `minimal` = UI tweens only, `normal` = + live video, `full` = + sheet cycling. Gates: `level().ui()/live()/sheets()`; `--no-anim` is a legacy alias for normal. The hovered tile gets a separate tile-size lane in a never-evicted atlas slot. All lanes seek-match the thumbnail's frame (10% duration via cached meta) and die on target move / focus loss. Background ffmpeg/ffprobe jobs run `nice -n 10` — user attention wins CPU.
- **Two players, one contract (`SeekablePlayer` / `LivePlayer`):** the selected stream + warm pool ride **`SeekablePlayer`** (`sb-media/src/seekable.rs`) — in-process libav via `rsmpeg` (PLAN.md §15, settled + benchmarked 2026-07): resident demuxer + VT session per stream, so `seek(target, exact)` is a demuxer jump + decoder flush (keyframe ~10–30ms; exact = decode-forward, GOP-bound) instead of the CLI era's ~1s respawn. The hover lane still uses the CLI-pipe **`LivePlayer`** (tile-size, never seeks). Both share the same contract: frames due-stamped into a small bounded queue (depth 3 — decode-ahead absorbs keyframe/cold-start spikes), `take_frame` surfaces a frame only once due, so presentation paces on the **render clock**; late frames **re-anchor** instead of accruing debt (debt = stutter-then-fast-forward). `LivePlayer` stamps 1/fps under CFR-forced `-r` output; `SeekablePlayer` stamps by **pts delta** from a wall-clock anchor (same degradation for wrong-meta/VFR: right wall-clock speed). Decode gates are identical, meta-driven: VT on macOS **only for h264/hevc/prores** (VP9/AV1 benchmarked *slower* through VT); **hardware scaling** (PERF.md — cut 4K60→1440p from ~3.7 to ~0.9 cores) for VT codecs with known 8/10-bit 4:2:0 `pix_fmt` via `scale_vt`+`hwdownload` at target res, dims rounded DOWN to mod-8, ±90° via `transpose_vt` (PSNR-verified) — `SeekablePlayer` reuses the exact `hw_scale_vf` string as its libavfilter graph, so the chains can't drift; anything else takes the sw chain (which `SeekablePlayer` rotates explicitly — libavfilter doesn't autorotate like the CLI; 90/180/270 covered). Missing `pix_fmt` (old cache) → sw chain now + queued reprobe heals next spawn. The bounded queue is what makes pre-warm free: an undrained player stalls its decoder via backpressure after ~4 frames. Drop MUST wake the condvar (`closed` + notify): a stalled player's reader parks on the full-queue wait where nothing else can reach it — without the wake every dropped warm player leaked a thread pinning ~30MB (live video degraded after minutes of browsing). Warm spawn order is a priority (serialized one at a time): right, right+1, down, left — no 'up' (rare); the second-right slot makes repeated right-taps instant. Regression tests: `live_player_paces_frames`, `unwatched_player_stalls_then_serves_instantly`, `dropped_player_releases_its_reader` + the same trio for seekable (`seekable::tests`) and `seek_jumps_in_place_without_respawn`.
- **Slot budget & eviction classes:** statics claim budget center-out, anims get the remainder; eviction order = out-of-zone anims → out-of-zone statics → in-zone anims; never in-zone statics or live slots.
- **Grid feel:** per-clip scale + emphasis springs (hover and keyboard ride the same morph: grid crop-fill → true-aspect cover, UV crop derives from live shape); anim sheets cycle by UV window with per-clip phase and shader crossfade (`anim_crossfade`, second UV per instance); reflow crossfade on zoom/resize; loading dots + cloud badges + background-jobs progress bar are all just extra tiles. The jobs bar draws **above** the quickview dim so "still working" stays visible in the modal.
- **Perf:** idle render throttling — app returns `Frame.animating`; the loop drops to a 100ms tick when false (~2% core), wakes on input. `a` toggles sheet animation at level full (generation is skipped entirely when sheets are off). **The loop must never rely on the vsync present as its only frame cap:** an occluded window's `render()` early-returns before presenting, so `ControlFlow::Poll` free-ran `frame()` at ~100% core the whole time the app was backgrounded (machine saturated by the time the user came back — looked like a resume freeze). `WindowEvent::Occluded` forces the idle tick regardless of `animating`, and the Poll path floors the redraw interval at `MIN_FRAME` (4ms, under any real refresh rate) as a runaway guard for any other skipped-present path.
- **Emphasized tiles never show anim sheets** (tiny crop-fill frames zoom badly under the morph and live lands on different content) — static thumb until live arrives.
- **Focus pause:** losing window focus stops live lanes + sheet cycling (grid stays, still) and lets the idle throttle engage; `pause_unfocused` config (default true), `p` toggles at runtime.
- **Skip (`[`/`]`):** an in-place `player.seek(target, exact=true)` on the resident decoder — the stream object, its timeline, and `first_frame` all survive; no respawn, no checkpoint machinery (the old `warm_skip`/`skip_delta` pre-warm dance is deleted). The new offset wraps mod duration; `SelLive::position()` is the decoder's real pts (the seek target while one is in flight — the bar shows where playback is *going*). `hires_shown` (path of the frame in the hires texture) keeps the last frame on screen through the GOP-bound decode-to-target (~30–600ms) — freeze-then-jump, not reload. A slim position bar flashes over the video (selected tile or quickview modal) for `SKIP_BAR_HOLD_S` then fades. Amount = `skip_fraction` tuning, or per-binding `amount` on an internal command. Skips are **exact** seeks (honest landings on sparse-keyframe sources); the future seekbar scrub uses keyframe mode while dragging + exact on release. Test: `skip_seeks_the_selected_stream_in_place`.
- **Siblings swap (`D`, `open_parent`):** rebuilds the library from the selected clip's parent dir (non-recursive `spawn_dir_reader`), streamed like any ingest. `pending_reselect` holds the clip's path across the swap: update_live neither reaps nor spawns by index while it's set (indices are churning; the shielded stream keeps playing — display gates match by **path**, not index), and when the path streams back in it becomes the selection again (cleared on ingest close if the file vanished). Old jobs keep counting; thumbs re-serve from the disk cache. Test: `open_parent_swaps_to_siblings_and_reselects`.
- **Next:** **M8 quickview scrub** (PLAN.md §14 M8 + §15 "Low-latency seek"). The prerequisite **in-process libav port is DONE** (spike benchmark in `spikes/seek-bench`, results table in §15: keyframe seeks 9–30ms, exact GOP-bound up to ~600ms, CLI respawn baseline 280–880ms; port landed as `SeekablePlayer`, see the players bullet above). Remaining M8 features: seekbar reveal/fade, click-to-seek (keyframe seeks while dragging, exact on release), storyboard hover thumbs, filmstrip hover-play + scroll; M7 search/filter; **edge-peek quickview** (deferred milestone: the ±1 neighbors' *videos* peek at the screen edges and slide in on advance — pre-warmed decoders are already the feed, but it needs the single hires texture to become a small array + slide choreography).

## Build & run
Building needs the **ffmpeg 8.x dev libraries** (brew `ffmpeg` ships them; `rsmpeg` links via pkg-config) on top of the ffmpeg/ffprobe CLIs the pipeline shells out to.
```sh
cargo run -- --demo                # demo grid: 480 fake tiles (no args + TTY stdin = help)
cargo run -- ~/Clips              # CLI paths (files or dirs; dirs walk when recurse=true)
fd -e mp4 -e mov . ~/Clips | cargo run    # or streamed over stdin (args win if both)
RUST_LOG=debug cargo run           # more logging
```
Keys: `hjkl`/arrows move selection (reserved, not remappable; horizontal moves are linear — row-end wraps to the next row) · `Enter`/`o` open in mpv · `Space` internal quickview · `[`/`]` skip back/fwd through the playing clip (`skip_fraction` of duration, wraps; a binding can carry its own `amount`) · `D` browse the selected clip's parent dir (siblings view; the playing clip keeps playing and re-selects itself) · `c` copy path · `r` reveal in Finder · `a` toggle anim · `p` toggle focus-pause · `f` fullscreen (the `fullscreen` internal action = macOS native/own Space; a separate `fast_fullscreen` action = mpv-style borderless desktop-sized window via winit `set_simple_fullscreen` — bind either or both; each exits whichever mode is active; `--fullscreen`/`--fast-fullscreen` start that way, queued as a `ToggleFullscreen{fast}` command drained in `resumed()` before the first frame) · `-`/`=`/`0` zoom · `q` quit. All non-movement keys remappable via `[keys]`/`[commands]` in `switchblade.toml` (`type = "launch"` programs with `{path}`/`{dir}`/`{name}` templates, or internal actions — see `commands.rs` for the list). Trackpad pans without changing selection; pinch zooms; click selects; clicking the selection quickviews. iCloud placeholder files get a blue tile + cloud badge and are never read.

## Layout
- `src/main.rs` — thin entrypoint.
- `crates/sb-window` — owns the winit loop + wgpu renderer; defines the `App` trait boundary. Instanced rounded-rect tiles in `tiles.wgsl`.
- `crates/sb-app` — implements `App`: grid model, selection, motion, stdin ingest, tuning hot-reload. Headless of OS/GPU types.
- `crates/sb-media` — worker pool (priority queue), ffmpeg/ffprobe invocation, sidecar cache, fingerprinting, `LivePlayer` (CLI pipe, hover lane) + `SeekablePlayer` (in-process libav, selected/warm lanes).

## Hard rules (distilled from PLAN.md)
- **Boundary:** no winit/wgpu types in `sb-app`. `sb-window` owns the loop and calls into the `App` trait; apps see normalized `InputEvent`s and return a plain `Frame`.
- **stdin is sacred:** streaming, newline + NUL delimited, never block the UI on a slow producer. File stats happen on the ingest thread, not the render path. CLI path args take priority over stdin; non-video extensions are skipped; dirs walk when `recurse` (default true). Probe meta records display rotation — consumers must swap decode dims for ±90° clips or portrait phone footage stretches.
- **No text stack until M7.** Labels are window-title or bitmap-font debug quality only.
- **Filesystem cache, no SQLite** (until profiling proves otherwise — PLAN.md §8).
- **Feel constants live in `Tuning`** (`crates/sb-app/src/tuning.rs`), hot-reloaded (mtime poll, 250ms) from the first existing of `./switchblade.toml` → `~/.config/switchblade.toml` → `~/.config/switchblade/config.toml` (`config_path()`; `--init` writes the user copy). Never hardcode a motion/feel value elsewhere — add a field.
- Motion must stay frame-rate independent: use `tuning::alpha(k, dt)` for smoothing, never raw per-frame lerps.
- Keep dependencies lean; ask before adding one.

## Conventions
- rustfmt defaults, edition 2021.
- `cargo build` from repo root builds everything; a cwd `switchblade.toml` beats the ~/.config ones, so running from repo root during development uses the repo config (which is also the `--init` template, embedded via include_str!).
